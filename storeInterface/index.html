Задание

Создать программу, которая принимает список товаров магазина в формате JSON (предположительно данные получены из Базы Данных). Функционал, который требуется реализовать:

  • a) Посмотреть список товаров
  • b) Установить фильтры
  • c) Сортировать товары
  • q) Выход из программы

Посмотреть список товаров:

При выборе пункта "Посмотреть список товаров", в консоль должен выводиться список товаров, подготовленных к выводу (дата должна выводиться в более привычном нам формате, к примеру: 31.12.2019, 23:59:59). Делать это лучше с помощью console.table, который принимает два аргумента: 1й - массив или объект, 2й - массив свойств, которые следует отображать. В нашей ситуации нужно просто передать туда массив товаров. Список товаров должен выводиться не всегда весь: если установлены фильтры (или же дополнительно указана сортировка), то нужно вывести только отфильтрованные товары. После списка должны также выводиться: количество, суммарная стоимость и средняя цена выведенных товаров. По желанию можно добавить самый дорогой товар и его цену, а также самый дешевый товар и его цену.

При выборе данного пункта, список главного меню должен показываться снова.

Установить фильтры:

При выборе пункта "Установить фильтры", пользователю должно поступить предложение выбрать тип фильтра: категория, цена, производитель, дата изготовления, сброс фильтров, а также возможность вернуться в основное меню.

Категория:

При выборе "Категория" пользователю должен поступить список категорий всех товаров. Каждый пункт должен быть представлен буквой для удобства работы с программой. Пользователь может выбрать один или несколько пунктов, написанных слитно. Дополнительно должен быть ещё один пункт в конце, который принимает все категории, которые выбрал пользователь, но он выбирает не эти категории, а все, кроме них. Это может быть реализовано так: a) Холодильник b) Телевизор c) Телефон ... -) Все, кроме. К примеру, пользователь вводит: bc-. Должны быть отброшены только товары b и c, а остальные переданы на вывод. Более удобный и практичный интерфейс приветствуется.

Цена:

При выборе "Цена" пользователю сначала должно быть предложено ввести верхний предел цены (максимальная цена) товаров, а затем - нижний (минимальная цена). При этом автоматически при этих предложениях должны быть указаны текущие минимум и максимум (при вводу максимальной цены - максимум, при минимальной - минимум), чтобы пользователь мог сразу согласиться с каким-либо из них. Опять же, для удобства.

Производитель:

При выборе "Производитель" функционал должен быть аналогичным "Категория".

Дата изготовления:

При выборе "Дата изготовления" пользователю должны по очереди поступить предложения указать минимальную и максимальную даты производства товара. В дате одновременно должны быть указаны только год и месяц.

Сброс фильтров:

При выборе "Сброс фильтров" абсолютно все фильтры должны быть сброшены.

После подтверждения выбора любого из этих пунктов, программа должна возвращать пользователя в главное меню, а в консоль выводить сообщение о проведённых установках. Однако список товаров выводиться не должен (для этого есть пункт "Посмотреть список товаров", а пункт "Установить фильтры" лишь занимается необходимой фильтрацией).

Сортировать товары:

При выборе пункта "Сортировать товары", пользователю должно поступить предложение выбрать критерий сортировки: категория, цена, производитель, дата изготовления, в обратном порядке, а также возможность вернуться в основное меню. Пользователь может выбрать любой из вариантов и дополнительно пункт "В обратном порядке". К примеру, это может выглядеть так: a) Категория b) Цена c) Производитель ... r) В обратном порядке. Пользователь может выбрать: ar. Любой из выбранных пунктов должен сортировать текущие товары, которые переданы на вывод в консоль, в зависимости от критерия.

Выход из программы:

При выборе пункта "Выход из программы" завершается выполнение всей программы.

Особенности:
  1. Предыдущие особенности остаются
  2. Обязательно воспользоваться продвинутыми методами массивов. Всё задание построено вокруг них
  3. Воспользоваться датами и временем
  4. Воспользоваться методами объекта JSON
Дополнительно:

Необязательным условием будет: добавить отображение всех установленных фильтров и режимов сортировки в главное меню, чтобы сразу было понятно какой ожидается вывод товаров.
